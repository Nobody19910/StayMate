🔴 CRITICAL ISSUES (Fix Immediately — Blocks Public Launch)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. /api/push-subscribe — No Auth
   ❌ Anyone can subscribe anyone to notifications
   ✅ Fix: Validate auth.getUser() before upsert
   ⏱️  30 min

2. /api/push-notify — No Authorization
   ❌ Anyone can send fake admin notifications
   ✅ Fix: Check profile.role === "admin"
   ⏱️  45 min

3. /api/upload-image — No Ownership Check
   ❌ Users can upload to other users' properties
   ✅ Fix: Verify property ownership before upload
   ⏱️  2 hours

4. Mock OTP "123456" in Production
   ❌ Anyone can log in with email + hardcoded OTP
   ✅ Fix: Remove client-side OTP, use Supabase Auth only
   ⏱️  15 min

5. Paystack Payment Not Verified
   ❌ Client-side "payment succeeded" → fraud possible
   ✅ Fix: Implement webhook verification in API route
   ⏱️  3 hours

   SUBTOTAL: ~6 hours to fix Critical issues


🟠 HIGH-SEVERITY ISSUES (Fix Before Production)

6. Disabled Auth Guards on Admin Routes
   ❌ Admin page was accessible without role check
   ✅ Fix: Re-enable useEffect auth guard
   ⏱️  5 min

7. No Rate Limiting
   ❌ Brute-force attacks, spam notifications
   ✅ Fix: Add Upstash Ratelimit to API endpoints
   ⏱️  1.5 hours

8. Weak RLS for Push Subscriptions
   ❌ "Service role reads all" policy exposes endpoints
   ✅ Fix: Restrict to admin + owner only
   ⏱️  30 min

9. No Audit Logging
   ❌ Can't detect fraud (admin accepts then rejects)
   ✅ Fix: Create audit_logs table + triggers
   ⏱️  2 hours

10. No CSRF Protection
    ❌ POST endpoints vulnerable to cross-site attacks
    ✅ Fix: Verify Origin header on API routes
    ⏱️  1.5 hours

    SUBTOTAL: ~6 hours to fix High issues
